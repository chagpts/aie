IFRS 15: Five-Step Revenue Recognition Model

Step 1: Identify the Contract

A contract exists when enforceable rights and obligations arise. This is usually established at order confirmation â€” not when a payment flow starts.

Step 2: Identify Performance Obligations

Each distinct good or service must be identified. Most Stripe use cases involve a single obligation satisfied at a point in time.

Step 3: Determine the Transaction Price

The transaction price is the amount expected to be received. Stripe fees and refunds affect presentation, not the transaction price itself.

Step 4: Allocate the Transaction Price

If multiple obligations exist, the price is allocated based on relative stand-alone selling prices.

Step 5: Recognize Revenue

Revenue is recognized when control transfers to the customer. This typically aligns with charge.created or charge.succeeded, not Stripe balance availability.

Key takeaway:
Revenue follows economic substance, not cash movement.

What Is an API? (In Plain English)

An API (Application Programming Interface) is a standardized way for one system to communicate with another system using predefined rules and data formats.

In simple terms:

An API is like a menu.
